## 1. Copywriting — 4/4
- 인라인 힌트 탁월: `"YouTube 링크 (여러 개는 줄바꿈)"`, `"최근 다운로드 (더블클릭 → 폴더 열기)"`.
- 결과 메시지 정보성: `"완료 — 2개 다운로드"`, `"완료 — 성공 2, 실패 1"`, `"취소됨 — 성공 N, 실패 M"`.
- 다운로드 중 파일명 노출(P3): `(1/3) My Video.mp4` — 무엇을 받는지 즉시 전달.
- ✅ 반영(2026-06-26): `Mp4/Mp3` → `MP4/MP3` 대문자로 통일(UI 라벨·README).

## 2. Visuals — 4/4  ▲ (3→4)
- **P3로 포맷/화질 박스 높이 불균형 해소**: `AddStretchSpacer`로 드롭다운을 박스 수직 가운데에 두고 두 박스를 `EXPAND`로 동일 높이 정렬 — 스크린샷에서 균형 확인.
- 전용 앱 아이콘(빨간 비디오+다운로드)으로 타이틀바/작업표시줄 식별성 확보.
- 주 액션 버튼 볼드+파란 기본버튼 테두리로 시각 무게 확보.
- 잔여(매우 사소): 다운로드 버튼 글리프 없음(텍스트 의존) — 네이티브 유틸리티엔 비필수.

## 3. Color — 4/4  ▲ (3→4)
- OS 네이티브 시스템 색 기반(데스크톱 적절).
- **P3로 상태 의미색 추가**: 성공=초록(`0,128,0`), 실패/오류=빨강(`180,0,0`), 진행/대기=시스템 기본색.
- 다음 실행 시작 시 `SYS_COLOUR_WINDOWTEXT`로 리셋 → 다크모드에서도 안전, 실패색 잔상 없음.
- `SetDefault` 파란 강조가 주 액션 의미색 역할.

## 4. Typography — 4/4  ▲ (3→4)
- 주 액션 볼드로 1차 위계 확보, `StaticBox` 제목이 그룹 라벨로 기능.
- **✅ 반영(2026-06-26)**: 섹션 헤더 라벨("YouTube 링크", "최근 다운로드")을 볼드 처리해 본문과 2차 위계 분리.
  네이티브 관례를 지키려 크기는 키우지 않고 굵기만 적용 — 다운로드 버튼과 동일한 `GetFont().Bold()` 패턴.

## 5. Spacing — 4/4
- 섹션 시작마다 `TOP 12`, 라벨↔필드 밀착, 좌우 거터 12px 일관, 상태줄 `TOP 4` 근접 — 의도된 리듬(P1).

## 6. Experience Design — 4/4
- 워커 스레드 + `wx.CallAfter`로 반응성 유지, 진행률 게이지 + 항목별 상태.
- **취소 경로 3-수렴(P3)**: 버튼 토글 · Esc(`EVT_CHAR_HOOK`) · 창닫기가 모두 같은 토글 경로로 수렴(중복 로직 없음).
- 접근성(P1): Enter 실행, `Alt+D/Alt+B`, 읽기전용 폴더 필드 키보드 탭 제외.
- 견고성(P2): 결과 요약, 최소 창 크기, 종료코드 3분기로 silent failure 제거.
- 진행 피드백(P3): 현재 파일명 표시 + 상태 의미색.
- 설정/히스토리 영속화, 더블클릭 폴더 열기, 종료 시 프로세스 정리(고아 방지).
